## 前端接口文档（生产可用）

统一返回结构：`Result<T>`

- code: number（200 成功）
- message: string
- data: T
- timestamp: number

说明：除非特别说明，所有接口均返回上述统一结构；下方“返回示例”中的最外层即为 `Result<T>`。

## 一、AI 对话与数据清洗聚合（`/api/ai`）

### 1) 清洗 + 查询 + 热度聚合

- 方法与路径：`POST /api/ai/clean-search`
- 功能：将用户自然语言清洗为查询条件，优先查缓存（课程候选）→未命中查数据库；识别“热门搜索/热度”意图并附带 Top5；返回聚合“可用数据”。
- Query：
  - `limit`（number，可选，默认 10，范围 1-50）课程返回条数上限
- Body（JSON）：
  ```json
  { "rawText": "最近大家都在搜 编程 龙洞 3学分 热门" }
  ```
- 返回数据结构：`AIAvailableDataDTO`
  - `cleanedParams`（`AIQueryParamDTO`）清洗后的结构化查询条件
  - `matchedCourses`（`CourseBase[]`）按条件查询的课程列表（已限条，可能来自缓存）
  - `hotIntent`（boolean）是否识别到“热度/热门搜索”意图
  - `hotKeywords`（`SearchKeyword[]`）当 `hotIntent=true` 时返回 Top5

- 返回示例：
  ```json
  {
    "code": 200,
    "message": "查询成功",
    "data": {
      "cleanedParams": {
        "courseName": "%编程%",
        "teacherName": null,
        "campus": "广东工业大学龙洞校区",
        "category": "十校公选",
        "courseTime": "周二第3-4节",
        "credit": "3.0",
        "scoreExpr": ">=4.0",
        "tags": "给分松,事少",
        "cleanedInput": "最近 大家 都 在 搜 编程 龙洞 3 学分 热门"
      },
      "matchedCourses": [
        {
          "courseId": "f45e34...49",
          "courseName": "Python编程基础",
          "teacherName": "李老师",
          "category": "十校公选",
          "campus": "广东工业大学龙洞校区",
          "credit": 3.0,
          "courseTime": "周二第3-4节",
          "courseSchedule": "全学期线下",
          "score": 4.6,
          "aiSummary": "……",
          "tag": "给分高,事少",
          "createTime": "2025-10-15T12:01:57",
          "updateTime": "2025-10-15T12:01:57"
        }
      ],
      "hotKeywords": [
        { "keywordId": 1, "type": "course_name", "value": "Python编程", "count": 212 },
        { "keywordId": 2, "type": "course_name", "value": "数据分析", "count": 175 }
      ],
      "hotIntent": true
    },
    "timestamp": 1734444444444
  }
  ```

### 2) 可用数据聚合（不触碰缓存）

- 方法与路径：`POST /api/ai/available-data`
- 功能：输出“可用数据”（清洗条件 + 课程候选 + 热度Top5），不做缓存读写逻辑；用于仅取数据、稍后前端自行渲染或传 AI。
- Query：`limit`（默认 10，1-50）
- Body：同上
- 返回：同 `AIAvailableDataDTO`（参考上一个接口示例）

### 3) 真实 AI 对话（前端会话 ID 驱动）

- 方法与路径：`POST /api/ai/real-chat`
- 功能：调用阿里百炼应用，将“可用数据 + 用户对话”合并为 Prompt 与会话 `sessionId` 一起交给 AI；无本地上下文缓存。
- Body（JSON）：
  ```json
  {
    "sessionId": "front-session-123",
    "userInput": "大家最近在搜什么编程课？推荐龙洞3学分的。"
  }
  ```
- 返回数据结构：`AIChatResponseDTO`
  - `aiResponse`（string）AI 文本回复
  - `contextId`（string）等于前端传入的 `sessionId`
  - `relatedCourses`（`CourseBase[]`）来自“可用数据”的课程候选
  - `relatedComments`（`CourseComment[]`）当前为空列表
  - `processingTime`（number）
  - `fromCache`（boolean）
- 返回示例：
  ```json
  {
    "code": 200,
    "message": "AI回复生成成功",
    "data": {
      "aiResponse": "最近大家都在搜Python编程基础...（AI文本）",
      "contextId": "front-session-123",
      "relatedCourses": [
        { "courseId": "f45e34...49", "courseName": "Python编程基础", "teacherName": "李老师", "score": 4.6, "tag": "给分高,事少" }
      ],
      "relatedComments": [],
      "processingTime": 0,
      "fromCache": false
    },
    "timestamp": 1734444444444
  }
  ```

### 4) 热门搜索 TopN

- 方法与路径：`GET /api/ai/hot-search`
- 功能：返回某类型热门搜索词的 TopN
- Query：
  - `type`：`course_name|teacher_name|campus|college|tag|category`（默认 `course_name`）
  - `limit`：number（默认 5，1-20）
- 返回示例：
  ```json
  {
    "code": 200,
    "message": "热门搜索获取成功",
    "data": [
      { "keywordId": 1, "type": "course_name", "value": "Python编程", "count": 212 }
    ],
    "timestamp": 1734444444444
  }
  ```

### 5) 仅清洗

- 方法与路径：`POST /api/ai/clean`
- 功能：将用户自然语言清洗为结构化参数，便于 UI 直接回显或用于后续查询。
- Body：`{ "rawText": "龙洞 3学分 给分高 周二3-4节" }`
- 返回示例：
  ```json
  {
    "code": 200,
    "message": "清洗成功",
    "data": {
      "courseName": "%编程%",
      "teacherName": "%张三%",
      "campus": "广东工业大学龙洞校区",
      "category": "十校公选",
      "courseTime": "周二第3-4节",
      "credit": "3.0",
      "scoreExpr": ">=4.0",
      "tags": "给分松,事少",
      "cleanedInput": "龙洞 3 学分 给分高 周二 3-4 节"
    },
    "timestamp": 1734444444444
  }
  ```

### 6) 辅助（可选）

- GET `/api/ai/context` → 返回 `AIChatContextDTO`
- POST `/api/ai/context`（Body: `AIChatContextDTO`）→ `boolean`
- GET `/api/ai/history` → `AIChatContextDTO`
- DELETE `/api/ai/history` → `boolean`
- GET `/api/ai/quick?question=…` → `string`
- GET `/api/ai/status` → `{ status, version, mode, timestamp }`

## 前端接入要点

1. `real-chat` 需传入 `sessionId` 维持与百炼的会话；后端按“可用数据 + 用户对话”构造 Prompt。
2. 模糊匹配：清洗后 `courseName` 作为通用关键词，OR 匹配课程名/教师名/标签，能覆盖“Python编程基础”与“编程”等部分词。
3. 热度：若输入包含“热门/热度/热搜/大家都在搜/最近在搜”等，`hotIntent=true` 且附带 `hotKeywords Top5`。
4. 缓存：`clean-search` 仅对课程列表读/写缓存；热门搜索每次实时查询。
5. 分页与返回大小限制：课程列表默认 10 条；部分接口限制 `limit≤50`（详见各节）。
