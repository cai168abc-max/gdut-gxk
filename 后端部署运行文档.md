# 广东工业大学课程评价系统 - 后端部署运行文档

## 1. 项目概述

广东工业大学课程评价系统是一个基于Spring Boot 3.x开发的后端服务，提供课程信息查询、评价管理、智能对话等功能。系统采用分层架构设计，集成了MySQL数据库、Redis缓存和阿里云大模型服务。

## 2. 系统架构

### 2.1 技术栈
- **后端框架**: Spring Boot 3.2.0
- **ORM框架**: MyBatis-Plus 3.5.5
- **数据库**: MySQL 8.0+
- **缓存**: Redis + Caffeine (二级缓存架构)
- **AI服务**: 阿里云通义千问大模型
- **工具库**: Hutool 5.8.25
- **构建工具**: Maven

### 2.2 分层架构
```
Controller层 (接口控制)
    ↓
Service层 (业务逻辑)
    ↓
Mapper层 (数据访问)
    ↓
Entity层 (数据模型)
```

## 3. 环境要求

### 3.1 基础环境
- **操作系统**: Windows/Linux/macOS
- **JDK**: 17或更高版本
- **Maven**: 3.6.0或更高版本
- **MySQL**: 8.0或更高版本
- **Redis**: 5.0或更高版本

### 3.2 硬件要求
- **CPU**: 2核心或以上
- **内存**: 4GB或以上
- **存储**: 10GB可用空间

### 5.2 配置数据库
1. 创建数据库表结构
```sql
-- 使用gdut_gxk数据库
USE gdut_gxk;

-- 创建课程表
CREATE TABLE course_base (
    course_id VARCHAR(32) PRIMARY KEY COMMENT '课程ID',
    course_name VARCHAR(255) NOT NULL COMMENT '课程名称',
    teacher_name VARCHAR(100) COMMENT '教师姓名',
    category VARCHAR(50) COMMENT '课程类别',
    campus VARCHAR(50) COMMENT '校区',
    college VARCHAR(100) COMMENT '学院',
    credit DECIMAL(3,1) COMMENT '学分',
    course_time VARCHAR(100) COMMENT '上课时间',
    arrangement VARCHAR(255) COMMENT '课程安排',
    rating DECIMAL(3,2) DEFAULT 0.00 COMMENT '评分',
    ai_summary TEXT COMMENT 'AI总结',
    tags VARCHAR(255) COMMENT '标签',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='课程基础信息表';

-- 创建评论表
CREATE TABLE course_comment (
    comment_id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '评论ID',
    course_id VARCHAR(32) NOT NULL COMMENT '关联课程ID',
    attendance_frequency VARCHAR(50) COMMENT '考勤频率',
    exam_type VARCHAR(50) COMMENT '考试形式',
    score TINYINT COMMENT '评分(1-5分)',
    content TEXT COMMENT '评论内容',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '提交时间',
    FOREIGN KEY (course_id) REFERENCES course_base(course_id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='课程评论表';

-- 创建搜索关键词表
CREATE TABLE search_keyword (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    keyword VARCHAR(100) NOT NULL COMMENT '搜索关键词',
    match_word VARCHAR(100) COMMENT '匹配词',
    type TINYINT COMMENT '类型(0-通用,1-校区,2-类别,3-标签)',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='搜索关键词表';

-- 创建索引
CREATE INDEX idx_course_name ON course_base(course_name);
CREATE INDEX idx_teacher_name ON course_base(teacher_name);
CREATE INDEX idx_category ON course_base(category);
CREATE INDEX idx_campus ON course_base(campus);
CREATE INDEX idx_college ON course_base(college);
CREATE INDEX idx_course_comment_course_id ON course_comment(course_id);
CREATE INDEX idx_search_keyword ON search_keyword(keyword);
```

### 5.3 配置应用
1. 修改配置文件 `src/main/resources/application.yml`:
```yaml
server:
  port: 8080

spring:
  application:
    name: gdut-gxk
  
  # 数据源配置
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/gdut_gxk?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    username: gdut_user
    password: your_password
    
  # Redis配置
  data:
    redis:
      host: localhost
      port: 6379
      password: 
      database: 0
      lettuce:
        pool:
          max-active: 8
          max-wait: -1ms
          max-idle: 8
          min-idle: 0
      timeout: 10000ms

# MyBatis-Plus配置
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: assign_id
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
  mapper-locations: classpath*:/mapper/**/*.xml

# 日志配置
logging:
  level:
    com.gdut.gxk: debug
    org.springframework.web: debug
  pattern:
    console: "%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"

# 管理端点配置
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics
```

## 6. API接口文档

### 6.1 课程相关接口
- **获取课程列表**: `GET /api/course/list`
  - 参数: keyword(关键词), category(类别), campus(校区), tags(标签), pageNum(页码), pageSize(页大小)
  - 返回: 课程列表分页数据

- **获取课程详情**: `GET /api/course/detail/{courseId}`
  - 参数: courseId(课程ID)
  - 返回: 课程详细信息及评论列表

- **搜索课程**: `GET /api/search/course`
  - 参数: q(搜索关键词)
  - 返回: 匹配的课程列表

### 6.2 评价相关接口
- **提交课程评价**: `POST /api/course/comment`
  - 参数: courseId(课程ID), attendanceFrequency(考勤频率), examType(考试形式), score(评分), content(评价内容)
  - 返回: 提交结果

- **获取课程评价列表**: `GET /api/course/comments/{courseId}`
  - 参数: courseId(课程ID), pageNum(页码), pageSize(页大小)
  - 返回: 评价列表分页数据

### 6.3 AI对话接口
- **AI对话**: `POST /api/ai/chat`
  - 参数: message(用户消息), sessionId(会话ID，可选)
  - 返回: AI回复内容

- **无状态AI对话**: `POST /api/ai/stateless/chat`
  - 参数: message(用户消息), contextId(上下文ID，可选)
  - 返回: AI回复内容

- **获取对话历史**: `GET /api/ai/history/{sessionId}`
  - 参数: sessionId(会话ID)
  - 返回: 对话历史记录

- **清除对话历史**: `DELETE /api/ai/history/{sessionId}`
  - 参数: sessionId(会话ID)
  - 返回: 清除结果

### 6.4 数据统计接口
- **获取热门搜索**: `GET /api/hot-search`
  - 参数: limit(返回数量，默认10)
  - 返回: 热门搜索关键词列表

- **获取数据概览**: `GET /api/summary/overview`
  - 返回: 课程数量、评价数量等统计数据

### 6.5 管理接口
- **更新课程ID**: `POST /admin/data/update-course-id`
  - 返回: 更新结果
  - 注意: 仅管理员可调用

## 7. 常见问题与解决方案

### 7.1 数据库连接问题
**问题**: 应用启动时提示数据库连接失败
**解决方案**:
1. 检查MySQL服务是否启动
2. 确认数据库用户名和密码是否正确
3. 检查防火墙设置，确保端口3306可访问
4. 确认数据库URL配置是否正确

### 7.2 Redis连接问题
**问题**: 应用启动时提示Redis连接失败
**解决方案**:
1. 检查Redis服务是否启动
2. 确认Redis配置是否正确
3. 检查防火墙设置，确保端口6379可访问

### 7.3 端口占用问题
**问题**: 应用启动失败，提示端口8080已被占用
**解决方案**:
1. 修改配置文件中的端口号
2. 或者终止占用8080端口的进程

### 7.4 内存不足问题
**问题**: 应用运行时出现内存不足错误
**解决方案**:
1. 增加JVM堆内存大小: `java -Xmx2g -jar target/gdut-gxk-0.0.1-SNAPSHOT.jar`
2. 优化应用代码，减少内存使用

### 7.5 AI服务调用失败
**问题**: AI对话功能无法正常工作
**解决方案**:
1. 检查阿里云通义千问API密钥是否正确配置
2. 确认网络连接是否正常
3. 检查API调用次数是否超限

### 8.1 数据库优化
1. 为常用查询字段添加索引
2. 定期分析表和优化表
3. 考虑读写分离架构

### 8.2 缓存优化
1. 合理设置缓存过期时间
2. 监控缓存命中率
3. 考虑使用分布式缓存集群

### 8.3 应用优化
1. 使用连接池管理数据库连接
2. 启用Gzip压缩减少网络传输
3. 考虑使用CDN加速静态资源访问
